### Detailed Description of Service Classes

#### `OrderService` (`com.example.orderservice.service.OrderService`)
The `OrderService` class is the heart of the `orderservice` microservice. It encapsulates all business rules related to order processing.

- **`createOrder(OrderRequest request)`**: Validates the incoming `OrderRequest`, maps it to an `Order` entity, persists it via `OrderRepository`, and publishes an `order-created` event to Kafka via `OrderEventProducer`. Returns an `OrderResponse`.
- **`getOrderById(Long id)`**: Retrieves an order by its ID. Throws `ResourceNotFoundException` if the order is not found. Maps the `Order` entity to `OrderResponse`.
- **`getAllOrders(String status, int page, int size)`**: Fetches a paginated list of orders. Can filter by `OrderStatus`. If no status is provided, retrieves all orders.
- **`updateOrder(Long id, OrderRequest request)`**: Updates an existing order. Throws `ResourceNotFoundException` if the order does not exist. Applies updates, saves the entity, and publishes an `order-updated` event.
- **`deleteOrder(Long id)`**: Deletes an order by its ID. Throws `ResourceNotFoundException` if not found. Publishes an `order-deleted` event.
- **`cancelOrder(Long id)`**: Attempts to cancel an order. Validates if the order is in a cancellable state. If successful, updates the status to `CANCELLED`, persists the change, and publishes an `order-cancelled` event. Throws `IllegalStateException` if the order cannot be cancelled (e.g., already shipped/delivered).
- **`isOrderOwnedByCustomer(Long orderId, Long customerId)`**: Checks if a specific order belongs to a given customer. Useful for authorization checks.
